# 猫咪呼吸监控系统 — 项目总结 v2.0

## 项目目标

搭建 7×24 小时猫咪呼吸频率监测系统：摄像头采集 → 网格光流检测 → 三方法共识 → 飞书告警。无需 YOLO。

## 最终架构

```
安卓手机(IP Webcam + Termux/frpc)
        ↓ frp TCP 穿透
腾讯云轻量服务器(Docker)
  ├── frps (接收 frpc 连接，映射 18080 端口)
  └── cat_monitor (每5分钟: 抓帧30s → 4×4网格光流 → 三方法共识 → CSV+飞书)
        ↓
飞书群告警
```

## 检测方法演进

| 版本 | 方法 | 检测率 | 准确性 |
|------|------|--------|--------|
| v1.0 | YOLOv8n + ROI 光流 | ~25%（猫蜷缩无法识别） | 12-14 bpm（采样率偏） |
| v1.2 | YOLOv8n + 峰计数+FFT | ~25% | 17→21 bpm（prominence 修复） |
| v2.0 | 网格光流 + 三方法共识 | ~90%+ | 22-25 bpm（与人工计数吻合） |

## 验证结果

| 指标 | 数值 |
|------|------|
| 帧率 | ~35 帧 / 30 秒（FRAME_SKIP=2，帧间隔 ~0.85s） |
| 网格 | 4×4，640px 缩放 |
| 呼吸频段 | 0.25-0.8 Hz（15-48 bpm） |
| 呼吸频率 | 22-25 次/分钟（人工计数 21-25，吻合） |
| 检测率 | >90%（不再依赖猫识别） |

## 关键技术

- **4×4 网格光流**：全帧 Farneback 光流分 16 格，找运动最低的 5 格 → FFT 扫描呼吸频段
- **网格锁定**：3 轮热身期后锁定最佳检测格，后续复用，SNR 衰减超 60% 才解锁
- **三方法共识**：峰计数（主）+ FFT（次）+ 自相关（辅）加权平均
- **谐波/次谐波抑制**：FFT >25 bpm 检查是否 2× 基频 → 零权重；10-17 bpm 检查 2× FFT 峰 → 自动校正
- **历史权重**：FFT 峰距近 4 次均值 ±5 bpm 内 → 1.5× 加成
- **bbox 中值平滑**：3 帧窗口平滑 YOLO 抖动，静止判定阈值 45%（v1.x 遗留）

## 依赖

- 安卓手机（IP Webcam + Termux + frpc）
- 云服务器（Ubuntu 22.04，Docker，≥1GB 可用内存）
- 飞书应用（App ID + Secret，群机器人权限）

## 已知限制

- frp 隧道偶发断连（运营商 CGNAT / Android 后台策略）
- 超强环境噪声下信号质量下降（ac_strength < 0.10 时拒绝）
- 网格锁定后如果猫换位置且原位置 SNR 不衰减，不会自动解锁（设计如此：保守策略）
